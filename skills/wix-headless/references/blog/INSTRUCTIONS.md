# Blog Implementer

Extends `references/shared/IMPLEMENTER.md`. Read that file first for phase routing, REST auth + doc lookups, prompt-inlined inputs (read only your `.wix/seeded.json` slice), return contract, style conventions, and common failure modes.

## Scope routing

| Scope | Phase | Reference |
|-------|-------|-----------|
| `seed` | Seed (post seeding via REST with Ricos content) | `./BLOG_CONTENT.md` |
| `components` | Components (blog service module, RicosViewer, consts, astro.config sitemap) | `../astro/blog/BLOG_SETUP.md` |
| `pages` | Pages (listing, post detail, category detail, RSS, BlogPost layout — under `src/pages/blog/*` + `src/pages/category/[slug].astro`; item pages register `wixMetadata` and render `<SEO.Tags>` from `@wix/seo`) | `../astro/blog/BLOG_PAGES.md` |

Cover images are generated by the shared image agent (`image-phase-2-entity` scope) AFTER `seed` completes — not by this agent.

## Files this vertical creates / contributes

The `components` and `pages` scopes enumerate their own file lists — see `../astro/blog/BLOG_SETUP.md` (components) and `../astro/blog/BLOG_PAGES.md` (pages).

## Blog-specific failure modes

| Wrong | Right |
|---|---|
| `import { items } from "@wix/data"` for blog queries | `import { posts } from "@wix/blog"` — use the Blog SDK |
| `<article set:html={post.content} />` | `<RicosViewer client:only="react" content={post.richContent} />` — content is Ricos format |
| Skip `wixMetadata` on `[...slug].astro` / `category/[slug].astro` | Required on **both** item pages for sitemap + dashboard SEO editor + deep links |
| Read `WIX_APPS` into a variable before `wixMetadata` | Reference `WIX_APPS` **directly** inside the export — module-scope eval; one throwing `wixMetadata` clears all of `/_wix/pages.json` |
| Hand-roll `<title>`/OG in `BaseHead` **and** render `<SEO.Tags>` | Pick one source per tag — `<SEO.Tags>` owns SEO on item pages; `BaseHead` keeps only non-SEO global metadata (see `BLOG_PAGES.md` §1) |
| Fetch `seoData` to feed the resolver | `loadSEOTagsServiceConfig` resolves from `itemType` + `slug` alone — don't pass `seoData` |
| Expect category SEO overrides to appear | Known Wix backend gap: category overrides don't resolve; page still gets default tags + sitemap entry (see `BLOG_PAGES.md` §1 caveats) |
| Query posts without `RICH_CONTENT` fieldset | Always request `RICH_CONTENT` or `post.richContent` is undefined |
| Scoped `<style>` for RicosViewer overrides | `<style is:global>` — React islands don't inherit scoped Astro styles |
